    Subversion Erişim Hakları Subversion Access Rights

    Merhaba, Hi,

    Ben subversion için bir newbie. I am a newbie to subversion.
    Ben bireysel kullanıcılara / gruplara ayrı erişim hakları vermek istiyorum. I want to give separate access rights to individual users/groups.

    Aşağıdaki gibi benim dizin yapısı: I my directory structure is as follows:

    Kod: Code: 
     / Proje / /project/
/ Proje / src / /project/src/
/ Proje / doc /project/doc
    Şöyle Benim authz dosyası: My authz file is as follows:

    Kod: Code: 
     [Grup] [groups]
sys-geliştiriciler = dev1, dev2 sys-developers = dev1, dev2
doc-yazar = Doc1, doc2, dev1, dev2 doc-writers = doc1, doc2, dev1, dev2
Herkes = dev1, dev2, Doc1, doc2 everyone = dev1, dev2, doc1, doc2

[/] [/]
@ Sys-geliştiriciler = rw @sys-developers = rw

[Marengo :/ proje / doc /] [marengo:/project/doc/]
@ Doc-yazar = rw @doc-writers = rw
@ Sys-geliştiriciler = r @sys-developers = r
    I / doc 'rw' / belge yazarlar için klasör ve 'src' klasörüne hiçbir erişim vermek istiyorum. I want to give 'rw' to /doc/ folder for document writers and no access to 'src' folder.
    Ben geliştiriciler için 'r' erişim ve 'doc' klasörüne hayır 'rw' erişim vermek istiyorum. I want to give 'r' access to developers and no 'rw' access to 'doc' folder.

    Ben sys-geliştiriciler giriş ile svn + ssh yaparsanız, ben tüm proje olsun (ve ben de doc rw olduğunu düşünüyorum, bu sorun değil) If I do svn+ssh with sys-developers login, I get the entire project (and I think there is rw to doc too, thats alright)

    Ama ben doktor-yazarlara verilecek 'src' okuma erişimi istiyor bilmiyorum. But I dont want read access to 'src' to be given to doc-writers.
    Geçerli authz yapılandırma çalışmak için görünmüyor ve ben doktor-yazar kullanarak giriş eğer bir verir The current authz configuration doesnt seem to work and if I login using doc-writers it gives a
    Kod: Code: 
     svn: Yetkilendirme başarısız oldu svn: Authorization failed
    Ben de okunabilir 'r' için doc-yazar erişim '/' iyi çalışıyor, ama 'src' verirsek, Bunun olmasını istemiyorum. If I give 'r' access to doc-writers for '/' it works fine, but 'src' can also be read, I dont want that to happen.

    Şimdi, ben istediğimi gerçekleştirmek ve yetkilendirme dosyası için yapılması gereken değişiklikler nelerdir yapabilirsiniz. Now, can I accomplish what I want and what are the changes to be done to authz file.

    Herhangi bir Yardım harika olurdu. Any help would be great.

    Teşekkür ederim, Thank You,
    Yaswanth Yaswanth
